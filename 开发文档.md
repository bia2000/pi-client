## **3. 核心 Agent 设计与工具封装**

Agent 的核心在于将招聘流程拆解为可被 LLM 调用的工具。当用户输入“帮我找5个北京3年经验的Vue前端，并打分”时，Agent 会自主执行以下闭环：

### **3.1 爬虫工具 (**`boss_spider_tool`**)**

- **机制**：为了防风控，不使用重型 Playwright 全页渲染，而是基于 Node.js 的 `axios` 配合用户登录态，直接请求 BOSS直聘的推荐列表接口。
- **防封策略**：工具内置随机延时（5-15秒）、随机 User-Agent 池。若触发风控（返回安全验证页面），工具返回特定错误码，Agent 接收后自动暂停任务并提醒用户手动滑块验证。
- **参数设计**：接收 `职位关键词`、`城市`、`页码` 等参数，返回结构化的候选人简要列表。

### **3.2 简历解析工具 (**`resume_parser_tool`**)**

- **机制**：接收爬虫返回的候选人 ID 或 HTML 片段，清洗掉页面冗余标签，提取核心字段（姓名、期望薪资、工作经历、项目经验、技能标签）。

### **3.3 智能评分工具 (**`resume_scorer_tool`**)**

- **机制**：调用本地或云端 LLM。将用户的岗位 JD 作为基准，与解析出的简历内容打包发送给 LLM。
- **Prompt 设计**：强制 LLM 以 JSON 格式输出评分结果，包含：`总分(0-100)`、`技能匹配度(高/中/低)`、`跳槽频繁度风险`、`推荐理由`。

### **3.4 数据库工具 (**`database_tool`**)**

- **机制**：封装 `better-sqlite3`。Agent 调用此工具将评分大于 60 分的候选人存入本地“牛人库”，支持后续的查询与状态更新（如：待沟通、已发offer）。

## **4. 前端界面全量重构 (Vue 3)**

**技术栈**：Vue 3 + Vite + TypeScript + Pinia + Vue Router + Element Plus (或 Naive UI)。

**核心页面与组件设计**：

- **Agent Console (智能体控制台)**：
  - 类似 ChatGPT 的对话界面，用户通过自然语言下达招聘指令。
  - 核心组件 `<AgentTimeline />`：以时间轴形式实时渲染 Agent 的思考链路（如：“正在调用爬虫获取第1页...”、“发现10份简历，正在解析...”、“评分完成，5人符合要求已入库”）。
- **Talent Pool (牛人库看板)**：
  - 核心组件 `<ResumeTable />`：展示已入库的候选人列表，支持按“评分”倒序排列，使用 Element Plus 的 Table 组件展示姓名、评分、核心技能。
  - 核心组件 `<ResumeDetail />`：点击列表项弹窗展示简历全文，并高亮 AI 标注的匹配点。
- **Settings (系统设置)**：
  - 配置 LLM API Key、BOSS直聘 Cookie 导入框、爬虫频率阈值设置。

## **6. 开发与构建工作流**

1. **初始化项目**：使用 `pnpm create @quick-start/electron`，选择 Vue + TypeScript 模板。
2. **后端工具开发**：在 `electron/main/tools` 目录下单独开发并测试爬虫、解析、评分工具的函数。
3. **Agent 编排**：在 `agentCore` 中引入工具，编写 System Prompt，使 LLM 能够根据用户指令正确选择并按序调用工具。
4. **前端联调**：运行 `pnpm dev`，在 Vue 界面中下发指令，通过 IPC 监听 Agent 的流式输出并更新界面。
5. **生产打包**：执行 `pnpm build:win` (或 mac/linux)，生成轻量级安装包。

## **7. 关键风控与避坑指南**

1. **登录态失效处理**：BOSS直聘的 Cookie 有效期较短。必须在 `spider.ts` 中加入对特定响应状态码（如重定向到登录页）的判断。一旦失效，需通过 IPC 通知 Vue 前端弹出“请重新导入 Cookie”的警告。
2. **爬虫频率极致控制**：自动化爬取极易触发账号封禁。Agent 在循环调用爬虫工具时，必须在工具内部强制 Sleep（如每次请求间隔 8-15 秒），且单次任务抓取量建议不超过 50 份。
3. **LLM 评分成本控制**：如果调用云端 API（如 GPT-4），大量简历评分会导致费用飙升。建议在 `scorer.ts` 中先做硬性规则过滤（如薪资期望不符直接淘汰），只将符合基础条件的简历交给 LLM 进行深度评分。
4. **隐私合规**：应用仅限本地使用，简历数据必须且只能存储在本地 SQLite，严禁在代码中植入任何向外部服务器发送候选人数据的逻辑。
